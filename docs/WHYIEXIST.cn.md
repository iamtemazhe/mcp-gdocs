# 为什么选择 mcp-gdocs

## 问题

Google Docs API 功能强大但很复杂。现有解决方案（面向 Google Docs 的 MCP 服务器）往往存在以下不足：

- 只覆盖基本读写，没有格式、表格或评论
- 缺少批量能力，迫使 AI 连续发起几十次调用
- 在混合插入与格式化时无法妥善处理索引冲突
- 每次格式化都要手动计算索引
- 把文档、表格、幻灯片捆在一个臃肿的服务器里

## mcp-gdocs 能为你做什么

### 1. 一个服务器掌控完整文档流程

**44 个工具**，覆盖 Google Docs 与 Drive 全流程，并支持多标签页文档：

**读取**

- 以纯文本、结构化数据或 Markdown 读取文档内容
- 获取文档元数据（标题、修订版本、语言区域）
- 列出文档中的标签页
- 可截断超大文档以节省上下文
- 轻量读取模式，只取所需数据

**写入**

- 插入、追加、删除与替换文本
- 替换整篇文档内容
- 全文查找与替换
- 插入分页符
- 重命名文档标签页

**格式**

- 文本样式（粗体、斜体、下划线、删除线、字体、字号、颜色）
- 段落样式（对齐、间距、缩进、段前分页）
- 标题样式（H1–H6 与正文）
- 按文本查找并套用格式——无需事先知道位置

**表格**

- 创建表格，增删行列
- 更新单元格内容与背景样式
- 一步创建并填入数据的表格

**图片**

- 从 URL 插入
- 从本机文件插入（自动上传并插入）

**Markdown**

- 用 Markdown 替换文档（标题、列表、表格、粗体/斜体、删除线、链接、代码块、引用、水平线、图片）
- 将 Markdown 追加到文档末尾
- 全文替换时保留文档标题
- 用第一个标题设置文档标题

**批量操作**

- 在单次请求中组合多种不同类型的操作
- 操作数量很大时自动分块

**评论**

- 列出、查看、创建、回复、解决与删除评论

**Google Drive**

- 列出、搜索、创建、复制、重命名、移动与删除文档和文件夹
- 从模板创建文档
- 查看文件夹内容与元数据
- 可控制 Drive 响应的详细程度

### 2. 处处支持批量

写入、格式、评论与 Drive 相关工具均支持单次调用传入多项。一次调用可完成几十项操作——例如一次请求内格式化大量标题、插入大量文本块，或在 Drive 上创建多个文件。

其他实现通常不支持这种「多项一次」的模式。

### 3. 按文字排版，而不是按坐标

在文档中查找文本并直接套用格式——不必先读结构再算位置。把常见的「读 → 找位置 → 排版」三步缩成一步。

### 4. Markdown 到 Google Docs——尽量完整

完整转换：标题、列表、表格、粗体/斜体、删除线、链接、代码块、引用、水平线与图片。分阶段执行：先完成插入再排版，减少位置冲突。

许多现有解决方案往往明确写明 Markdown 表格和图片「尚不支持」。

### 5. 插入本地图片

从磁盘读取图片、上传到 Drive、设置权限并插入文档——一步完成。

### 6. 安全的段落样式

自动调整位置，缓解 Google Docs API 中常见的「段落样式串到相邻段落」问题。

### 7. 五种认证方式

OAuth、服务账号（文件路径）、Base64 凭证（适用于 Docker/CI）、应用默认凭证（适用于 GKE/Cloud Run）以及用户模拟（impersonation）。支持多配置文件，隔离令牌存储。其他实现通常只支持两种方式。

### 8. 可靠性

- 针对限流与服务器错误自动重试，带指数退避
- 全局与单次读取均可截断响应以节省上下文
- 尽量只请求所需数据的轻量 API 调用
- 结构化日志，日志级别可配置
- 错误信息清晰，并附带可操作的修复提示

## 优势

| 优势 | 说明 |
|------|------|
| 单次多项操作 | 各工具均支持数组，可批量执行 |
| 统一批量接口 | 单次调用可含多种操作类型 |
| Markdown 表格与图片 | 完整转换，含表格与图片 |
| 分阶段执行请求 | 先插入后格式——减少位置冲突 |
| 标题样式 | 独立的 H1–H6 工具 |
| 表格单元格样式 | 按单元格设置背景色 |
| 安全段落样式 | 自动位置调整 |
| 本地图片插入 | 上传与插入一步完成 |
| 5 种认证 | OAuth、SA、Base64、ADC、模拟用户 |
| API 错误重试 | 限流与服务器错误时自动重试 |
| 响应控制 | 全局截断、读取上限、Drive 详细度 |
| 结构化日志 | 可配置级别便于排查 |
