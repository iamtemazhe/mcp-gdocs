# Почему mcp-gdocs существует

## Проблема

Google Docs API мощный, но сложный. У существующих решений (MCP‑серверов для Google Docs) обычно такие недостатки:

- Охватывают только базовые операции (чтение/запись) без форматирования, таблиц и комментариев
- Нет пакетных возможностей — приходится делать десятки последовательных вызовов
- Не учитывают конфликты индексов при смешивании вставок и форматирования
- Требуют вручную считать индексы для каждой операции форматирования
- Объединяют Документы и Презентации в один перегруженный сервер

## Что даёт mcp-gdocs

### 1. Полный контроль над документом в одном сервере

**63 инструмента** для Google Docs, Drive и комментариев, с поддержкой нескольких вкладок в документе. MCP **resources** отдают недавние документы (`gdocs://documents/recent`); **prompts** — готовые сценарии (pretty-format, gost-format, template-fill, export-to-markdown):

**Чтение**

- Содержимое документа в виде текста, структурированных данных или Markdown
- Метаданные документа (название, ревизия, язык)
- Список вкладок документа
- Сокращение очень больших документов, чтобы экономить контекст
- Лёгкие режимы чтения — только нужные данные

**Запись**

- Вставка, дописывание в конец, удаление и замена текста
- Замена всего содержимого документа
- Поиск и замена по всему документу
- Разрывы страниц
- Переименование вкладок документа
- Текст верхнего и нижнего колонтитула (`docs_update_header`, `docs_update_footer`)
- Экспорт документа в PDF по URL (`docs_export_pdf`)

**Форматирование**

- Стили текста (жирный, курсив, подчёркивание, зачёркивание, шрифт, размер, цвет)
- Стили абзаца (выравнивание, интервалы, отступы, разрыв страницы перед абзацем)
- Стили заголовков (уровни 1–6 и обычный текст)
- Форматирование по найденному фрагменту — найти текст и применить стиль, не зная заранее позиций

**Таблицы**

- Создание таблиц, добавление и удаление строк и столбцов
- Обновление содержимого ячеек и фона ячеек
- Создание таблицы с данными за один шаг

**Изображения**

- Вставка по URL
- Вставка с локального диска (загрузка и вставка в одном действии)

**Markdown**

- Замена содержимого документа из Markdown (заголовки, списки, таблицы, жирный/курсив, зачёркивание, ссылки, блоки кода, цитаты, горизонтальные линии, изображения)
- Дописывание Markdown в конец документа
- Сохранение заголовка документа при полной замене
- Задание заголовка документа по первому заголовку в разметке

**Пакетные операции**

- Несколько разнотипных операций в одном запросе
- Автоматическое разбиение на части при очень большом числе операций

**Комментарии**

- Просмотр списка, чтение, создание, ответы, пометка как решённых и удаление

**Google Drive**

- Просмотр списка, поиск, создание, копирование, переименование, перемещение и удаление документов и папок
- Создание документов по шаблонам
- Просмотр содержимого папок и метаданных
- Настраиваемая подробность ответов для операций с Drive
- Общие диски (Shared Drives): `supportsAllDrives` и `includeItemsFromAllDrives` в операциях списка и поиска на Drive

### 2. Пакетность повсюду

Инструменты записи, форматирования, комментариев и Drive принимают несколько элементов за вызов, где это уместно. Один вызов может выполнить десятки операций — например, отформатировать много заголовков, вставить много фрагментов текста или создать несколько файлов на Drive за один запрос.

У других реализаций для Google Docs такой многоэлементной модели обычно нет.

### 3. Форматирование по тексту, а не по координатам

Можно найти текст в документе и сразу применить к нему оформление — без предварительного чтения структуры и подсчёта позиций. Типичный сценарий «прочитать → найти позицию → отформатировать» сокращается до одного шага.

### 4. Markdown в Google Docs — по максимуму

Полноценное преобразование: заголовки, списки, таблицы, жирный/курсив, зачёркивание, ссылки, блоки кода, цитаты, горизонтальные линии и изображения. Сначала выполняются вставки, затем форматирование — так реже возникают конфликты позиций.

У многих существующих решений таблицы и картинки в Markdown часто прямо отмечены как «пока не поддерживаются».

### 5. Вставка локальных изображений

Чтение файла с диска, загрузка в Drive, настройка доступа и вставка в документ — всё в одном действии.

### 6. Безопасное оформление абзацев

Автоматическая корректировка позиции снижает типичную проблему API Google Docs, когда стиль «перетекает» на соседние абзацы.

### 7. Четыре способа аутентификации

OAuth, сервисный аккаунт (путь к файлу, JWT, impersonation) и учётные данные в Base64 (для Docker и CI). Поддержка нескольких профилей для раздельного хранения токенов. У других реализаций обычно два способа входа.

### 8. Надёжность

- Автоматические повторы с нарастающей задержкой при лимитах и ошибках сервера
- Ограничение параллелизма через семафор (`GDOCS_MAX_CONCURRENT`) для сдерживания одновременных вызовов API
- Общее и пошаговое сокращение ответов, чтобы экономить контекст
- Запросы, которые по возможности запрашивают только нужные данные
- Структурированные журналы с настраиваемым уровнем детализации
- Понятные сообщения об ошибках с подсказками, что исправить

## Сильные стороны

| Преимущество | Подробности |
|--------------|-------------|
| Несколько операций за вызов | Массивы элементов в инструментах записи, форматирования, комментариев и Drive |
| Единый пакетный API | До 10 типов операций Docs в одном запросе |
| MCP resources и prompts | Недавние документы; готовые сценарии промптов |
| Shared Drives | Учёт общих дисков в списке и поиске на Drive |
| Колонтитулы и PDF | Текст верхнего/нижнего колонтитула; экспорт Docs в PDF по URL |
| Ограничение параллелизма | Опционально `GDOCS_MAX_CONCURRENT` (семафор) |
| Таблицы и картинки в Markdown | Полная поддержка таблиц, изображений, блоков кода, цитат |
| Пошаговое выполнение запроса | Сначала вставки, потом стили — меньше конфликтов позиций |
| Стили заголовков | Отдельные уровни H1–H6 и сброс в обычный текст |
| Стиль ячеек таблицы | Цвет фона по ячейке |
| Безопасные стили абзацев | Автокоррекция позиции для предотвращения перетекания стилей |
| Локальные изображения | Загрузка и вставка одним действием |
| 4 способа входа | OAuth, SA (JWT + impersonation), Base64 |
| Повтор при ошибках API | Автоматически при лимитах и сбоях |
| Управление ответами | Общее усечение, лимит на чтение, подробность Drive |
| Журналы | Настраиваемый уровень для отладки |
